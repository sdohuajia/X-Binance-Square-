# X → Binance Square Auto Sync

<p align="center">
  <img src="icons/icon-128.png" width="128" height="128" alt="X → Binance Square Auto Sync">

<p align="center">
  一个无需付费 X API 的 Chrome / Edge 扩展。<br>
  在 X 网页发布推文后，自动把文字同步到用户自己的币安广场。
</p>

<p align="center">

## 功能简介

用户只需在扩展设置页填写自己的 **币安广场 OpenAPI Key**。设置完成后，通过安装了本扩展的桌面浏览器发布新的 X 推文，扩展就会自动将推文文字发布到用户自己的币安广场。

工作流程：

```text
用户在 x.com 发布推文
  → X 返回发布成功结果
  → 扩展提取该条推文的 ID、作者和最终文字
  → 清理图片/视频附带的媒体短链接
  → 使用用户本地保存的币安广场 OpenAPI Key
  → 直接调用币安广场官方发布接口
```

本扩展不使用付费 X API，也没有开发者中转服务器。

---

## 支持范围

### 会自动同步

- 新发布的原创推文文字；
- 引用推文中用户自己编写的文字；
- 如果币安拒绝发布，插件图标显示红色数字角标；打开插件可查看错误码、拒绝原因、时间和推文摘要；
- 插件保留最近 3 条被拒绝记录；点击“立即重试”时，如果被拒内容含有链接，会先提醒确认，然后删除链接、仅发送剩余文字；
- 正文中用户主动添加的普通链接（优先按原文发布；如果币安返回 URL 安全风控 `20041`，扩展会自动移除链接并重试一次）；
- 从安装了本扩展的桌面 Chrome / Edge 发布的内容。

### 不会自动同步

- 回复推文；
- 直接转推；
- 图片和视频文件；
- 图片或视频附带的末尾 `t.co` 媒体短链接；
- 从 X 手机 App 发布的推文；
- 从其他未安装本扩展的浏览器发布的推文；
- 安装扩展之前已经发布的历史推文。

> 本扩展是文字同步工具。X 推文带图片或视频时，只同步干净文字，不会把无意义的媒体 `t.co` 链接发布到币安广场。

# 普通用户安装教程

## 第一步：下载扩展

1. 打开本仓库右侧的 **Releases**；
2. 点击最新版本，例如 `Latest`；
3. 在该版本的 **Assets** 中下载扩展 ZIP 安装包；
4. 不要直接双击运行 ZIP，需要先解压。

建议把 ZIP 解压到一个固定目录，例如：

```text
D:\X-to-Binance-Square-Extension\
```

或：

```text
C:\Users\你的用户名\Documents\X-to-Binance-Square-Extension\
```

> 安装后不要删除或移动这个解压目录，否则 Chrome / Edge 可能无法继续加载扩展。

解压后，请确认所选文件夹内能够直接看到：

```text
manifest.json
service-worker.js
page-hook.js
options.html
icons/
```

如果解压后出现两层同名目录，请选择真正包含 `manifest.json` 的内层文件夹。

---

## 第二步：安装到 Chrome

1. 在 Chrome 地址栏打开：

   ```text
   chrome://extensions/
   ```

2. 打开页面右上角的 **开发者模式**；
3. 点击左上角的 **加载已解压的扩展程序**；
4. 选择刚才解压、且内部直接包含 `manifest.json` 的文件夹；
5. 安装成功后，浏览器会显示 **X → Binance Square Auto Sync**；
6. 建议点击浏览器工具栏的扩展按钮，将本扩展固定到工具栏。

---

## 第三步：安装到 Microsoft Edge

如果使用 Edge：

1. 在 Edge 地址栏打开：

   ```text
   edge://extensions/
   ```

2. 打开左侧或页面中的 **开发人员模式**；
3. 点击 **加载解压缩的扩展**；
4. 选择包含 `manifest.json` 的扩展目录；
5. 安装成功后，将扩展固定到工具栏。

Chrome 和 Edge 的后续设置方法相同。

# 配置币安广场 OpenAPI Key

## 第一步：准备正确的 Key

本扩展需要的是：

```text
币安广场 OpenAPI Key
```

不是以下密钥：

- 不是币安现货交易 API Key；
- 不是币安合约交易 API Key；
- 不是钱包私钥；
- 不是助记词；
- 不是 X API Key。

请从币安广场官方提供的 OpenAPI/创作者工具入口获取发布专用 Key。不同账号或地区显示的入口可能不同，请以币安广场当前官方页面和文档为准。

> 严禁把交易 API Key、钱包私钥或助记词填写到扩展中。

## 第二步：填写并保存

1. 安装扩展后，扩展通常会自动打开设置页；
2. 如果设置页没有打开：
   - 打开 `chrome://extensions/` 或 `edge://extensions/`；
   - 找到本扩展；
   - 点击 **详情**；
   - 点击 **扩展程序选项**；
3. 将自己的币安广场 OpenAPI Key 粘贴到：

   ```text
   新增或替换 OpenAPI Key
   ```

4. 保持 **启用自动同步** 已勾选；
5. 点击 **保存设置**。

保存成功后：

- 输入框会立即清空；
- 页面只显示：

  ```text
  ● 已安全保存（内容已隐藏）
  ```

- 以后重新打开设置页，也不会把完整 Key 显示出来；
- 如需更换 Key，输入新 Key 后再次保存；
- 如需删除 Key，点击 **删除密钥**。

> “内容已隐藏”表示扩展不会把已保存密钥重新写回设置页面，并不代表硬件级加密。请保护好自己的电脑和浏览器用户目录。

# 绑定 X 账号

设置页首次显示：

```text
尚未绑定；首次成功检测到推文时自动绑定
```

这是正常状态，不需要手动输入 X 用户名，也不需要额外进行 X OAuth 授权。

自动绑定步骤：

1. 确保币安广场 OpenAPI Key 已保存；
2. 确保 **启用自动同步** 已勾选；
3. 使用当前浏览器登录自己的 X 账号；
4. 打开：

   ```text
   https://x.com/
   ```

5. 安装或更新扩展后，按一次：

   ```text
   Ctrl + Shift + R
   ```

   强制刷新 X 页面；
6. 使用该浏览器发布一条新的原创推文或引用推文；
7. X 返回发布成功后，扩展会自动识别作者并绑定，例如：

   ```text
   @your_username
   ```

8. 该条推文会同时进入币安广场发布流程。

自动绑定的作用是防止用户在同一个浏览器中切换 X 账号后，把其他账号的推文误发到当前币安广场。

如果以后需要更换 X 账号：

1. 打开扩展设置页；
2. 点击 **重置 X 账号绑定**；
3. 登录新的 X 账号；
4. 刷新 `x.com`；
5. 发布一条新的测试推文；
6. 扩展会自动绑定新账号。

# 日常使用方式

完成安装、密钥配置和首次绑定后，无需重复操作。

以后只需要：

1. 使用安装了本扩展的桌面 Chrome / Edge；
2. 登录已经绑定的 X 账号；
3. 在 `x.com` 正常发布原创推文或引用推文；
4. 等待 X 显示发布成功；
5. 扩展自动把干净文字发布到币安广场。

点击浏览器工具栏中的扩展图标，可以查看：

- 是否已配置 OpenAPI Key；
- 当前扩展是否启用；
- 已绑定的 X 账号；
- 最近一次同步是否成功；
- 待重试任务数量；
- 同步成功后的币安广场帖子链接。

网络临时失败时，扩展会把任务保存在本地队列并自动重试。也可以在扩展弹窗中点击 **立即重试**。

7. 返回 `x.com`；

# 常见问题排查

## 1. 发推后没有同步

请按顺序检查：

1. 是否使用安装了扩展的桌面浏览器发布；
2. 是否从 `x.com` 网页发布，而不是手机 App；
3. 扩展是否处于启用状态；
4. 币安广场 OpenAPI Key 是否显示“已安全保存”；
5. 是否勾选 **启用自动同步**；
6. 安装或更新扩展后，是否刷新过 `x.com`；
7. 当前 X 账号是否与已绑定账号一致；
8. 发布类型是否为原创或引用推文，而不是回复或直接转推；
9. 点击扩展图标，查看最近同步状态和错误提示；
10. 打开扩展设置后点击保存，再返回 X 强制刷新并发一条新的测试推文。

已经错过的旧推文不会自动补发。扩展只捕获安装并连接成功之后的新发推成功事件。

## 2. 一直显示“尚未绑定 X”

这表示扩展尚未捕获到一次符合规则的新发推成功事件。

处理方法：

1. 保存币安广场 OpenAPI Key；
2. 确认已启用自动同步；
3. 打开并登录 `x.com`；
4. 强制刷新 X 页面；
5. 发布一条新的原创测试推文；
6. 发布成功后再打开设置页查看绑定状态。

## 3. 显示账号不匹配

说明浏览器当前登录的 X 账号与扩展之前绑定的账号不同。

如果确实需要换号：

1. 打开设置页；
2. 点击 **重置 X 账号绑定**；
3. 确认当前 X 登录账号正确；
4. 刷新页面；
5. 发布一条新的测试推文完成重新绑定。

## 4. 带图片的推文出现 `t.co` 链接

最新版会在确认推文包含媒体时：

- 不上传图片或视频；
- 删除媒体附带的末尾 `t.co` 链接；
- 保留用户主动写在正文中的普通链接。

如果仍出现媒体短链接，请确认安装的是最新 Release，并在更新后强制刷新 X 页面。

## 5. 扩展安装后没有显示图标

1. 打开扩展管理页；
2. 确认扩展已启用；
3. 点击浏览器工具栏的扩展按钮；
4. 找到本扩展并点击固定；
5. 如果仍显示旧图标，重新加载扩展或重启浏览器。

## 6. 币安接口报错

可能原因包括：

- 填写的不是币安广场 OpenAPI Key；
- Key 已过期、被撤销或没有发布权限；
- 币安广场内容风控拒绝该文本；
- 请求频率过高；
- 当前网络无法访问币安；
- 币安广场官方接口发生变化。

请先删除旧 Key，再重新填写有效的币安广场发布 Key。

# 隐私与安全

## OpenAPI Key 如何保存

- 保存在当前浏览器扩展的 `chrome.storage.local`；
- 不使用 `chrome.storage.sync`；
- 不通过 Chrome 账号同步到其他设备；
- 不发送到开发者服务器；
- 只用于请求币安广场官方发布接口；
- 保存后不会重新显示在设置页 DOM 中；
- 用户可以随时删除；
- 卸载扩展会删除该扩展的本地数据。

浏览器本地存储不是硬件安全模块。如果设备、浏览器用户目录或扩展环境已经被恶意软件控制，本地数据仍可能泄露。请只使用权限最小化的币安广场发布 Key。

## 扩展读取哪些 X 数据

扩展只处理用户主动发布推文后，X 返回的成功 `CreateTweet` 响应，并提取：

- 新推文 ID；
- 当前 X 用户名；
- 最终推文文字；
- 公开推文链接；
- 用于判断是否有媒体的必要字段。

这些数据只用于当前自动同步任务。

## 扩展不会读取什么

本扩展不申请、不读取、不复制也不传输：

- 剪贴板内容；
- 复制或粘贴记录；
- 键盘输入记录；
- X 密码；
- X Cookie；
- X 私信；
- 浏览历史；
- 下载内容；
- 钱包私钥或助记词；
- 币安现货/合约交易数据；
- 与成功发推无关的 X 网络请求体。

## 权限范围

扩展权限仅包括：

```json
["storage", "alarms"]
```

网站访问范围仅包括：

```text
https://x.com/*
https://twitter.com/*
https://www.binance.com/*
```

本扩展不使用：

- `<all_urls>`；
- `clipboardRead`；
- `clipboardWrite`；
- `cookies`；
- `history`；
- `downloads`；
- `tabs`；
- `webRequest`；
- `debugger`；
- `nativeMessaging`；
- 远程 JavaScript；
- 开发者中转服务器。

完整说明请查看：

- [`PRIVACY.md`](PRIVACY.md)
- [`SECURITY.md`](SECURITY.md)

# 开发者说明

## 项目架构

```text
page-hook.js
  └─ MAIN world 中仅观察 CreateTweet 成功响应

bridge.js
  └─ 将最小化的发推结果桥接给扩展环境

service-worker.js
  ├─ X 账号本地绑定
  ├─ 本地队列和推文 ID 去重
  ├─ 失败自动重试
  └─ 直接调用币安广场发布接口

options.html / options.js
  ├─ OpenAPI Key 配置
  ├─ 保存后清空并隐藏
  ├─ 启用/停用同步
  └─ 重置 X 账号绑定

popup.html / popup.js
  └─ 显示配置和最近同步状态
```

## 本地加载源码

1. 克隆或下载源码；
2. 不要在源码中填写真实 OpenAPI Key；
3. 打开 `chrome://extensions/`；
4. 开启开发者模式；
5. 点击 **加载已解压的扩展程序**；
6. 选择源码根目录，即包含 `manifest.json` 的目录。

## 运行检查

项目不依赖打包器。安装 Node.js 和 Python 3 后，在源码目录运行：

```bash
node --check page-hook.js
node --check bridge.js
node --check service-worker.js
node --check options.js
node --check popup.js
node test-extension.js
node test-options.js
python3 audit_security.py
```

检查内容包括：

- 原创推文捕获；
- 引用推文捕获；
- 回复过滤；
- 无关 X 请求体不读取；
- X 账号自动绑定；
- 跨账号拦截；
- 推文 ID 去重；
- OpenAPI Key 保存后清空；
- 已保存 Key 不回填 DOM；
- 敏感权限扫描；
- 远程代码扫描；
- 私人密钥和开发者后端残留扫描。

# 已知限制

- 只支持安装了扩展的桌面 Chrome / Edge；
- 不支持捕获手机 App 发布的推文；
- 不会补发安装前或断开连接期间错过的推文；
- 当前只同步文字，不上传图片和视频；
- X 内部 `CreateTweet` 数据结构改变时可能需要更新；
- 币安广场接口或风控规则改变时可能需要适配；
- 如果币安已经成功发布，但浏览器在收到成功响应前断网，自动重试理论上可能造成重复，因为当前发布接口没有可用的客户端幂等键。
